üèóÔ∏è RDBMS Fundamentals
Relational Database Management System (RDBMS) ‚Äî A system that stores data in structured tables with predefined relationships
Core Concepts
Tables: Structured data containers with rows and columns
Rows: Individual records (tuples) containing data
Columns: Attributes defining the structure
Primary Keys: Unique identifiers for each record
Foreign Keys: Establish relationships between tables
ACID Properties
Atomicity: All operations succeed or fail together
Consistency: Database remains in valid state
Isolation: Concurrent transactions don't interfere
Durability: Committed changes persist

‚ûï INSERT - Create Data
-- Single row insert INSERT INTO employees (employee_id, first_name, last_name, salary, hire_date) VALUES (1, 'John', 'Doe', 75000, '2024-01-15');
-- Multiple rows insert (formatted for readability) INSERT INTO employees (employee_id, first_name, last_name, salary, hire_date) VALUES (2, 'Jane', 'Smith', 82000, '2024-02-01'), (3, 'Bob', 'Johnson', 68000, '2024-02-15');

üîç SELECT - Read Data
-- Basic selection SELECT employee_id, first_name, last_name, salary FROM employees; -- With WHERE condition SELECT * FROM employees WHERE salary > 70000 AND department_id = 1; -- With sorting and limit SELECT * FROM employees WHERE is_active = 1 ORDER BY salary DESC LIMIT 10;
‚úÖ Use LIMIT for pagination: LIMIT 20 OFFSET 40 (page 3, 20 items per page)
‚úèÔ∏è UPDATE - Modify Data
-- Update specific records UPDATE employees SET salary = salary * 1.1 WHERE hire_date < '2023-01-01'; -- Update with multiple conditions UPDATE employees SET department_id = 2 WHERE last_name = 'Smith';
‚ö†Ô∏è Always use WHERE clause with UPDATE! Without it, ALL rows will be modified.
üóëÔ∏è DELETE - Remove Data
-- Delete specific records DELETE FROM employees WHERE employee_id = 5; -- Delete with WHERE clause DELETE FROM employees WHERE hire_date < '2020-01-01' AND is_active = 0; -- Safer alternative - TRUNCATE (faster for large tables) TRUNCATE TABLE employees;
üí° DELETE vs TRUNCATE:
‚Ä¢ DELETE logs each row deletion (slower, can be rolled back)
‚Ä¢ TRUNCATE deallocates all pages (faster, cannot be rolled back in MySQL)

-- Create customer tracking system CREATE TABLE customers ( customer_id INT PRIMARY KEY AUTO_INCREMENT, customer_name VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE, registration_date DATE, total_purchases DECIMAL(12, 2) DEFAULT 0, is_premium BOOLEAN DEFAULT FALSE ); -- Add sample data INSERT INTO customers (customer_name, email, registration_date, total_purchases) VALUES ('Alice Johnson', 'alice@example.com', '2024-01-10', 5000), ('Bob Smith', 'bob@example.com', '2024-01-15', 3200); -- Query active customers SELECT * FROM customers WHERE registration_date >= DATE_SUB(NOW(), INTERVAL 30 DAY); -- Update customer status UPDATE customers SET is_premium = TRUE WHERE total_purchases > 5000;

-- Create server metrics table CREATE TABLE server_metrics ( metric_id INT PRIMARY KEY AUTO_INCREMENT, server_name VARCHAR(50) NOT NULL, cpu_usage DECIMAL(5, 2), memory_usage DECIMAL(5, 2), disk_usage DECIMAL(5, 2), recorded_at DATETIME DEFAULT CURRENT_TIMESTAMP, status VARCHAR(20) -- 'healthy', 'warning', 'critical' ); -- Insert monitoring data INSERT INTO server_metrics (server_name, cpu_usage, memory_usage, disk_usage, status) VALUES ('web-server-01', 45.2, 72.1, 85.3, 'warning'), ('db-server-01', 28.5, 88.9, 92.1, 'critical'); -- Query problematic servers SELECT * FROM server_metrics WHERE status IN ('warning', 'critical') ORDER BY recorded_at DESC LIMIT 10;
