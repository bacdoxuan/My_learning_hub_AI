üéØ JSONB Operators

Operator Description Example
-> Get JSON object field or array element data->'name'
->> Get text value data->>'name'
@> Contains (check if left contains right) data @> '{"status":"active"}'
<@ Contained by '{"a":1}' <@ data
? Key exists data ? 'name'
?& All keys exist data ?& ARRAY['a','b']
?| Any key exists data ?| ARRAY['a','b']

Operator Examples

-- Extract and filter SELECT * FROM products WHERE data->'category' = '"electronics"'::jsonb; -- Contains operator (most useful) SELECT * FROM products WHERE data @> '{"in_stock":true}'::jsonb; -- Key existence SELECT * FROM products WHERE data ? 'discount'; -- Combine multiple conditions SELECT data->>'name' FROM products WHERE (data->>'price')::NUMERIC > 100 AND data @> '{"brand":"Samsung"}'::jsonb;

üîß JSONB Functions

Key JSONB Functions
jsonb_set(): Update specific key
jsonb_array_elements(): Expand array elements
jsonb_each(): Expand object key-value pairs
jsonb_keys(): Get all keys
jsonb_agg(): Aggregate into JSON
json_build_object(): Construct JSON
jsonb_to_recordset(): Convert to rows

Function Examples

-- Update JSON value SELECT jsonb_set(data, '{price}', '99.99'::jsonb) FROM products WHERE id = 1; -- Expand array to rows SELECT jsonb_array_elements(data->'tags') FROM products; -- Build JSON from columns SELECT json_build_object( 'id', id, 'name', name, 'email', email ) FROM users; -- Aggregate rows to JSON array SELECT jsonb_agg(json_build_object( 'name', name, 'age', age )) FROM users;

üîç GIN Indexing for JSONB

-- Create GIN index on JSONB column CREATE INDEX idx_products_data ON products USING GIN (data); -- GIN index optimizes these operators: @>, @<, ?, ?&, ?| CREATE INDEX idx_data_contains ON products USING GIN (data jsonb_path_ops); -- Verify index usage with EXPLAIN EXPLAIN (ANALYZE) SELECT * FROM products WHERE data @> '{"brand":"Sony"}'::jsonb;

üìç JSON Path Queries (PostgreSQL 12+)

-- Use @? for path existence check SELECT * FROM products WHERE data @? '$.specifications.color'; -- Use @@ for more complex queries SELECT * FROM products WHERE data @@ '$.price < 1000'; -- jsonpath function SELECT jsonb_path_query_array(data, '$.*.price') FROM products;

‚úÖ Learning Checklist

  • Always use JSONB over JSON for performance
  • Master the @ operator family for queries
  • Index JSONB columns with GIN for performance
  • Use jsonb_set for updates
  • Extract array elements with jsonb_array_elements
  • Build JSON with json_build_object
  • Combine with EXPLAIN to verify index usage
  • Use JSON Path queries for complex filtering
  • Leverage GIN indexing for @> operator
üí° Pro Tip: JSONB with GIN indexes provides flexible schema storage with near-relational query performance. Perfect for evolving schemas or semi-structured data alongside relational tables.
