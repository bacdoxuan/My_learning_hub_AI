üèÜ Ranking Functions

ROW_NUMBER, RANK, DENSE_RANK

Function Description Example Output Use Case
ROW_NUMBER() Sequential numbers, no ties 1, 2, 3, 4, 5... Pagination, unique rankings
RANK() Standard ranking with gaps for ties 1, 2, 2, 4, 5... Competitions, leaderboards
DENSE_RANK() Consecutive ranking, no gaps 1, 2, 2, 3, 4... Dense rankings, categories
-- Ranking sales by amount SELECT ProductName, SalesAmount, ROW_NUMBER() OVER (ORDER BY SalesAmount DESC) AS RowNum, RANK() OVER (ORDER BY SalesAmount DESC) AS SalesRank, DENSE_RANK() OVER (ORDER BY SalesAmount DESC) AS DenseRank FROM ProductSales; -- Partitioned ranking by category SELECT CategoryName, ProductName, SalesAmount, ROW_NUMBER() OVER (PARTITION BY CategoryName ORDER BY SalesAmount DESC) AS CategoryRank FROM ProductSales ps JOIN Categories c ON ps.CategoryID = c.CategoryID;

‚¨ÖÔ∏è‚û°Ô∏è Offset Functions: LAG & LEAD

Accessing Previous/Next Rows

LAG Function
Access previous rows: LAG(column, offset, default)
Compare to prior periods: Month-over-month changes
Calculate differences: Growth rates, deltas
Default value: Handles first rows gracefully
LEAD Function
Access future rows: LEAD(column, offset, default)
Predictive analytics: Next period projections
Compare to future values: Trends, forecasts
Default value: Handles last rows gracefully
-- Month-over-month sales comparison SELECT SalesMonth, SalesAmount, LAG(SalesAmount, 1, 0) OVER (ORDER BY SalesMonth) AS PrevMonthSales, SalesAmount - LAG(SalesAmount, 1, 0) OVER (ORDER BY SalesMonth) AS MonthlyChange, CASE WHEN LAG(SalesAmount, 1, 0) OVER (ORDER BY SalesMonth) = 0 THEN 0 ELSE (SalesAmount - LAG(SalesAmount, 1, 0) OVER (ORDER BY SalesMonth)) / LAG(SalesAmount, 1, 0) OVER (ORDER BY SalesMonth) * 100 END AS GrowthPercent FROM MonthlySales ORDER BY SalesMonth; -- LEAD for next period prediction SELECT EmployeeName, CurrentSalary, LEAD(CurrentSalary, 1) OVER (PARTITION BY Department ORDER BY HireDate) AS NextPromotionSalary, LEAD(HireDate, 1) OVER (PARTITION BY Department ORDER BY HireDate) AS NextPromotionDate FROM EmployeeHistory;

üìä Running Totals & Aggregates

Accumulating Values with Window Frames

Running Totals Pattern
SUM() OVER: Cumulative sums with ORDER BY
ROWS UNBOUNDED PRECEDING: From start to current row
Partitioning: Separate running totals per group
Performance: Efficient with proper indexing
-- Running total of sales SELECT OrderDate, SalesAmount, SUM(SalesAmount) OVER (ORDER BY OrderDate ROWS UNBOUNDED PRECEDING) AS RunningTotal, AVG(SalesAmount) OVER (ORDER BY OrderDate ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS Rolling7DayAvg FROM DailySales ORDER BY OrderDate; -- Running total by category SELECT CategoryName, ProductName, SalesAmount, SUM(SalesAmount) OVER (PARTITION BY CategoryName ORDER BY SalesAmount DESC) AS CategoryRunningTotal FROM ProductSales ps JOIN Categories c ON ps.CategoryID = c.CategoryID;

üèùÔ∏è Gaps & Islands Problems

Finding Sequences and Missing Values

Gaps (Missing Sequences)
Identify missing values: Dates, IDs, sequences
LAG/LEAD comparison: Find breaks in continuity
Business applications: Missing transactions, gaps in inventory
Reporting: Data quality analysis
Islands (Consecutive Sequences)
Group consecutive values: Sessions, streaks, runs
ROW_NUMBER differences: Identify sequence groups
Business applications: User sessions, consecutive sales days
Analytics: Pattern recognition
-- Gaps: Find missing order dates WITH DateSequence AS ( SELECT DISTINCT CAST(OrderDate AS DATE) AS OrderDate FROM Orders WHERE OrderDate >= '2023-01-01' ), DateGaps AS ( SELECT OrderDate, LAG(OrderDate, 1) OVER (ORDER BY OrderDate) AS PrevDate, DATEDIFF(DAY, LAG(OrderDate, 1) OVER (ORDER BY OrderDate), OrderDate) AS DaysGap FROM DateSequence ) SELECT * FROM DateGaps WHERE DaysGap > 1; -- Islands: Group consecutive login days WITH LoginSequence AS ( SELECT UserID, LoginDate, ROW_NUMBER() OVER (ORDER BY LoginDate) AS RowNum, ROW_NUMBER() OVER (PARTITION BY UserID ORDER BY LoginDate) AS UserRowNum FROM UserLogins ), IslandGroups AS ( SELECT UserID, LoginDate, RowNum - UserRowNum AS IslandGroup FROM LoginSequence ) SELECT UserID, IslandGroup, MIN(LoginDate) AS StartDate, MAX(LoginDate) AS EndDate, COUNT(*) AS ConsecutiveDays FROM IslandGroups GROUP BY UserID, IslandGroup HAVING COUNT(*) > 2;
üí° Performance Best Practices: Always include ORDER BY in window functions. Use PARTITION BY to limit window size. Index columns used in PARTITION BY and ORDER BY clauses for better performance.
‚ö†Ô∏è Common Mistakes: Forgetting ORDER BY creates undefined results. Using window functions in WHERE clauses requires subqueries. Large window frames can be expensive - consider alternatives for very large datasets.
