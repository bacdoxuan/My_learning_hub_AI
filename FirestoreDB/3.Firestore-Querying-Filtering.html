Querying & Filtering

Master queries, filters, and pagination

üîç Simple Queries
Filter documents using where() clauses:
const q = query( collection(db, "users"), where("age", ">", 18) ); const snapshot = await getDocs(q);
Common operators: ==, <, >, <=, >=, !=, in, array-contains, array-contains-any
üîó Compound Queries
Combine multiple filters with AND logic:
const q = query( collection(db, "users"), where("age", ">", 18), where("status", "==", "active"), where("tags", "array-contains", "vip") );
‚ö†Ô∏è OR queries require a composite index. Consider OR queries as multiple queries instead
üìä Ordering & Limits
Sort and limit query results:
const q = query( collection(db, "users"), where("status", "==", "active"), orderBy("createdAt", "desc"), limit(10) ); const snapshot = await getDocs(q);
Order: asc (ascending) or desc (descending)
üìë Pagination with Cursors
Navigate through large result sets efficiently:
// First page const first = query( collection(db, "users"), orderBy("createdAt"), limit(25) ); const snapshot1 = await getDocs(first); const lastDoc = snapshot1.docs[snapshot1.docs.length - 1]; // Next page const next = query( collection(db, "users"), orderBy("createdAt"), startAfter(lastDoc), limit(25) );
Cursor methods: startAt(), startAfter(), endAt(), endBefore()
üîÑ Collection Group Queries
Query subcollections across all parent documents:
// Query all "posts" subcollections const q = collectionGroup(db, "posts"); const snapshot = await getDocs(q);
‚ö†Ô∏è Requires a collection group index to be enabled in Firestore
