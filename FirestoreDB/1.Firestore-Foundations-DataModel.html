Master the NoSQL document-oriented structure

üì¶ Collections & Documents
Firestore is a NoSQL, document-oriented database. Instead of tables and rows, data is organized into:
Collections: Containers holding documents
Documents: Individual records with key-value pairs
Fields: Keys and values within a document
// Document structure users/user123 { first: "Ada", last: "Lovelace", born: 1815 }
üí° Collections and documents are created implicitly when you add data
üå≥ Subcollections & Hierarchies
Documents can contain subcollections, allowing hierarchical data structures up to 100 levels deep:
users/userId/posts/postId { title: "Hello World", timestamp: 1640000000 }
Benefits: Better query organization, atomic updates at document level
