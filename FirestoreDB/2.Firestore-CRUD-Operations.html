CRUD Operations

Create, Read, Update, and Delete operations

‚úçÔ∏è Create Operations
Two main methods for adding data:
// Auto-generates document ID const docRef = await addDoc( collection(db, "users"), { first: "Ada", last: "Lovelace", born: 1815 } ); console.log(docRef.id);
// Use custom document ID await setDoc( doc(db, "users", "user123"), { first: "Ada", last: "Lovelace" } );
üí° set() with merge: Use {merge: true} to avoid overwriting existing data
üìñ Read Operations
Retrieve data with get() or real-time listeners:
// Get all documents const querySnapshot = await getDocs( collection(db, "users") ); querySnapshot.forEach((doc) => { console.log(doc.id, "=>", doc.data()); });
// Get specific document const docSnap = await getDoc( doc(db, "users", "user123") ); if (docSnap.exists()) { console.log(docSnap.data()); }
‚ôªÔ∏è Update Operations
Modify existing documents without replacing them:
await updateDoc( doc(db, "users", "user123"), { age: 30, "address.city": "London" } );
‚ö†Ô∏è updateDoc() only updates specified fields, unlike set() which replaces entire document
üóëÔ∏è Delete Operations
Remove documents or fields:
await deleteDoc( doc(db, "users", "user123") );
await updateDoc( doc(db, "users", "user123"), { age: deleteField() } );
üì¶ Batch Operations
Perform multiple writes atomically:
const batch = writeBatch(db); // Set, update, or delete multiple docs batch.set(doc(db, "users", "user1"), {...}); batch.update(doc(db, "users", "user2"), {...}); batch.delete(doc(db, "users", "user3")); await batch.commit();
Batch limits: Up to 500 writes per batch operation
‚ö° Transactions
Ensure multiple reads and writes succeed together:
await runTransaction(db, async (transaction) => { const docSnap = await transaction.get(docRef); const newValue = docSnap.data().value + 1; transaction.update(docRef, { value: newValue }); });
‚ö†Ô∏è Transactions can be retried. Keep transaction logic idempotent
