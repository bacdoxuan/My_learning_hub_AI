Security & Rules

Secure your database with Firestore Security Rules

üîê Basic Rules Structure
Firestore Security Rules follow a hierarchical match structure:
service cloud.firestore { match /databases/{database}/documents { match /users/{userId} { allow read, write: if request.auth.uid == userId; } } }
üîì Authentication Checks
Use request.auth to verify user identity:
// Only authenticated users allow read: if request.auth != null; // Specific user allow write: if request.auth.uid == userId; // Email verification allow read: if request.auth.token.email_verified;
Available auth fields: uid, token (claims), email, email_verified, etc.
üë• Role-Based Access Control
Use custom claims for fine-grained permissions:
// Admin-only access allow write: if get(/databases/$(database) /documents/users/$(request.auth.uid)) .data.role == "admin"; // Custom claim check allow read: if request.auth.token.role == "editor";
üìù Data Validation
Validate data format before writes:
allow create: if request.resource.data.keys() .hasOnly(["name", "email", "age"]) && request.resource.data.name is string && request.resource.data.email is string && request.resource.data.age is number;
